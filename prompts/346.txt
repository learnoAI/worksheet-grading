**TASK:**
Extract every number sequence completion question and its corresponding student answers from the worksheet. The questions are structured as vertical columns of boxes (Q1-Q4 on the first page, Q5-Q8 on the second) that you must reconstruct into horizontal sequences.

**JSON SCHEMA ENFORCEMENT:**
Respond with a JSON array of objects. Use EXACTLY these keys and follow the specified data types:
`"question_number"`, `"question"`, `"student_answer"`

**FORMATTING RULES:**
1. **question_number**:
   - MUST be a strict integer (e.g., `1`, `2`, `3`).
   - DO NOT output strings or prefixes (e.g., NO "Q1.", NO "1.").

2. **question**:
   - MUST be a string starting with the question label (e.g., `"Q1. "`, `"Q2. "`).

   - Substitute the missing/handwritten boxes with a single underscore `_`.
   - Separate every item in the sequence with a comma and a space (`, `).
   - Example format: `"Q1. 91, 92, _, _, _, _, _, _, 99, 100"`

3. **student_answer**:
   - MUST be a string containing ONLY the handwritten numbers filled into the blanks by the student.
   - Read the handwritten answers from top to bottom.

   - DO NOT use JSON arrays, brackets `[ ]`, or stringified arrays.
   - DO NOT include empty string placeholders `""` for pre-printed boxes. Just list the handwritten numbers in order.
   - Example format: `"93, 84, 95, 96, 97, 98"`

**HANDWRITING EXTRACTION GUIDELINES:**
- Look closely at digit formation, as the student's handwriting may be messy.
- Distinguish carefully between **9** vs **8**, **6** vs **0**, and **1** vs **7** vs **4**. Pay attention to visual stroke evidence (e.g., an intended 95 might look like 45, or an intended 96 might look like 90).

**PROCESSING ORDER:**
- Output the JSON array sorted by `question_number` ascending.
