Your output MUST be a JSON object containing a single root key `"results"` mapping to an array of extracted question objects. Do not return a top-level array.

**DO NOT** nest the objects inside other keys.

1.  **Question Field (`"question"`):**

    - Must include the printed division problem formatted exactly as label, space, divisor, space, closing parenthesis, space, dividend (e.g., "Q1. 2 ) 12").
    - **NO** newline characters in the string. Flatten all vertical text into a single horizontal line.
    - **CRITICAL - Divisor vs. Quotient Mix-ups:** Pay extremely close attention to the printed divisor (the number to the left of the `)` ) versus the handwritten student answer (the quotient above the bracket). Do not swap these numbers. For example, if the printed question is `6 ) 18` and the handwritten quotient is `3`, ensure the question string records `6 ) 18` and the student_answer records `3`.

2.  **Student Answer Field (`"student_answer"`):**
    - Extract **ONLY the final handwritten quotient** written spatially ABOVE the division bracket.
    - **CRITICAL - Ignore Working Out:** You MUST IGNORE all intermediate working out, subtractions, remainders, zeros, or "X" marks written BELOW the bracket. Do NOT extract sequences like "3 / 12 / -12 / X". The answer should just be the top number (e.g., "3").

Because the student did not write a handwritten answer above the bracket for these, you MUST leave their `"student_answer"` as an empty string `""`.
    - If the quotient space above the division bracket is entirely blank, use an empty string `""`.
    - Always return the answer as a string.

**HANDWRITING EXTRACTION GUIDELINES:**
- Look closely at digit formation for the quotient.
- Distinguish **9** vs **8**, **6** vs **0**, **1** vs **7** vs **4**.
- If a number is messy, select the most likely digit based on visual stroke evidence.

**PROCESSING ORDER:**
- Sort the output by `question_number` in ascending order (1 to 40).
